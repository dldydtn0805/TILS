# 코드

```c++
#include <iostream>
#include <vector>

using namespace std;

int N, M;
int trees[1000000];

long long cut_trees(int height)
{
    long long cnt = 0;
    for (int i=0; i<N; ++i) {
        long long tmp = trees[i] - height;
        if (tmp > 0) {
            cnt += tmp;
        }
    }
    
    return cnt;
}

int set_height()
{
    int start = 0;
    int end = 1000000000;
    
    while (start <= end) {
        long long mid = (start + end) / 2;
        long long firewood = cut_trees(mid);
        
        if (firewood >= M) {
            start = mid + 1;
        }
        else {
            end = mid - 1;
        }
    }
    
    return end;
}

int main()
{
    ios_base::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);
    
    cin >> N >> M;
    int num;
    for (int i=0; i<N; ++i) {
        cin >> trees[i];
    }

    cout << set_height();

    return 0;
}

```



> ## 문제
>
> 상근이는 나무 M미터가 필요하다. 근처에 나무를 구입할 곳이 모두 망해버렸기 때문에, 정부에 벌목 허가를 요청했다. 정부는 상근이네 집 근처의 나무 한 줄에 대한 벌목 허가를 내주었고, 상근이는 새로 구입한 목재절단기를 이용해서 나무를 구할것이다.
>
> 목재절단기는 다음과 같이 동작한다. 먼저, 상근이는 절단기에 높이 H를 지정해야 한다. 높이를 지정하면 톱날이 땅으로부터 H미터 위로 올라간다. 그 다음, 한 줄에 연속해있는 나무를 모두 절단해버린다. 따라서, 높이가 H보다 큰 나무는 H 위의 부분이 잘릴 것이고, 낮은 나무는 잘리지 않을 것이다. 예를 들어, 한 줄에 연속해있는 나무의 높이가 20, 15, 10, 17이라고 하자. 상근이가 높이를 15로 지정했다면, 나무를 자른 뒤의 높이는 15, 15, 10, 15가 될 것이고, 상근이는 길이가 5인 나무와 2인 나무를 들고 집에 갈 것이다. (총 7미터를 집에 들고 간다) 절단기에 설정할 수 있는 높이는 양의 정수 또는 0이다.
>
> 상근이는 환경에 매우 관심이 많기 때문에, 나무를 필요한 만큼만 집으로 가져가려고 한다. 이때, 적어도 M미터의 나무를 집에 가져가기 위해서 절단기에 설정할 수 있는 높이의 최댓값을 구하는 프로그램을 작성하시오.
>
> ## 입력
>
> 첫째 줄에 나무의 수 N과 상근이가 집으로 가져가려고 하는 나무의 길이 M이 주어진다. (1 ≤ N ≤ 1,000,000, 1 ≤ M ≤ 2,000,000,000)
>
> 둘째 줄에는 나무의 높이가 주어진다. 나무의 높이의 합은 항상 M보다 크거나 같기 때문에, 상근이는 집에 필요한 나무를 항상 가져갈 수 있다. 높이는 1,000,000,000보다 작거나 같은 양의 정수 또는 0이다.
>
> ## 해설
>
> 이 문제는 보자마자 저번에 풀었던 문제와 비슷한 문제라고 느꼈다. 바로 파라메틱 서치였다.
>
> 설정하려는 절단기의 높이를 찾는 이분탐색을 진행한다. 절단기의 최대 높이는 주어지는 나무의 최대 높이와 같으면 만족할 것이므로 1,000,000,000으로 한다. 모든 나무를 뿌리까지 잘라야하는 경우가 있기 때문에 최소값은 0으로 설정하여 이분탐색을 시작한다.
>
> > 1. start = 0, end = 1,000,000,000으로 설정한다.
> >
> > 2. `start <= end`를 만족하는 동안 `mid = (start + end) / 2`를 반복하여 현재 구간의 mid값을 구한다.
> >
> > 3. 현재 mid 값을 절단기의 높이로 설정했을 때 구할 수 있는 장작의 총 높이를 구한다.
> >
> > 4. 만약 장작의 높이가 M보다 작다면, 더 많은 양의 장작을 얻기 위해 `end = mid -1` 을 하고
> >
> >    만약 장작의 높이가 M보다 크거나 같다면, 환경을 위해 최대한 나무를 덜 잘라야 하므로, `start = mid + 1`를 한다.
> >
> > 5. start > end가 되었을 때, start = end = mid 상황에서 start가 앞으로 1칸가므로, end를 반환한다.
>
> 위의 방법을 토대로 문제를 풀었더니 정답을 맞았다..!
>
> 그러나! 한번 또 틀렸다 저번에 파라메틱 서치를 풀떄랑 동일한 이유였는데, 자료형을 int로 선언한 것이 화근이었다.
>
> 사실 오버플로우가 안날 것이라고 생각해서 int를 쓰긴했다. 바로 높이의 구간이 10억이 최대라서 start + mid가 int의 범위안에 들어가므로 문제가 없을 것이라 생각했으나, 나무를 자르고 얻는 장작의 높이의 총합이 int의 범위를 벗어날 수 있는 조건이었던 것이다...
